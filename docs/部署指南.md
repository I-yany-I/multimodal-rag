# 多模态 RAG 上线部署指南

本项目默认是 **本地 GPU 开发机**（CLIP + FAISS + Qwen2-VL + 可选 Whisper）。**上线**一般指：把同一套代码放到 **带 NVIDIA 驱动的 Linux 服务器** 或 **支持 GPU 的托管平台**，对外暴露 **HTTPS + 鉴权**，而不是长期打开 `share=True`。

## 方案对比（怎么选）

| 方式 | 适合 | 注意 |
|------|------|------|
| **自有云主机（腾讯云/阿里云/GCP 等）+ Docker** | 可控、可接域名与备案 | 需自行装 NVIDIA 驱动、`nvidia-container-toolkit`，显存建议 **≥12GB**（2B VLM + CLIP + whisper base 更宽裕） |
| **Hugging Face Spaces（GPU）** | 演示、作品集 | 冷启动、配额与 **模型权重拉取** 需按平台文档配置；Secrets 存 API/Token |
| **仅 CPU 主机** | 不推荐完整链路 | VLM 与 whisper 会非常慢或跑不动；只适合拆成「仅 API 检索 + 云端大模型」的改版 |

以下以 **Docker + 云主机** 为主流程（与是否腾讯云无强绑定）。

## 1. 准备清单

- Linux x86_64，**NVIDIA 驱动** 与 **CUDA** 与镜像内 PyTorch 版本大致匹配。
- 服务器安装 **Docker**、**Docker Compose**、**nvidia-container-toolkit**（使 `docker run --gpus all` 可用）。
- 域名（可选）+ **HTTPS**（Let's Encrypt 或云厂商证书）。
- 首次在服务器上跑通：`build_index.py` / `build_personal_index.py`，把 `data/` 里索引与图片按你隐私策略同步（大文件可用对象存储挂载，本仓库未内置）。

## 2. 使用仓库内 Docker 骨架

在项目根目录：

```bash
docker compose build
docker compose up -d
```

默认会尝试拉起 **API（8000）**。若你更希望对外只暴露 Gradio，可把 `docker-compose.yml` 里的 command 改成 `python app.py`，并改端口映射；生产环境更推荐 **API + 独立前端**，Gradio 仅作内网演示。

**环境变量（示例）**

- `HF_HOME` / `TRANSFORMERS_CACHE`：模型缓存目录，避免容器内重复下载。
- 不要在镜像里写死 **API Key**；用 compose `env_file` 或 K8s Secret。

## 3. 生产化最小增强（建议你自己逐步加）

1. **鉴权**：API Key、JWT 或网关层（nginx + auth_request）。  
2. **限流与超时**：防止大上传拖垮 GPU；`client_max_body_size`、FastAPI `TimeoutMiddleware` 或反向代理超时。  
3. **健康检查**：已有 `/health`，负载均衡器可配 readiness。  
4. **日志与请求 ID**：便于排障。  
5. **模型与索引版本**：环境变量写入 `index_manifest`，发布时 **模型+索引** 同步回滚。

## 4. Hugging Face Spaces（简要）

1. 新建 Space，选 **Docker** 或 **Gradio** 模板。  
2. 将本仓库推送到 Space 关联的 Git 仓库；在 Space 设置里打开 **GPU**（若可用）。  
3. 在 **Settings → Repository secrets** 配置 `HF_TOKEN`（若拉 gated 模型）。  
4. `README` 中写明启动命令与 **首次建索引** 是否在 `pre_build` 完成（大索引不建议每次构建重建）。

## 5. 我（助手）能替你做到哪一步？

- **已在仓库内提供**：Dockerfile / compose 骨架、本文档、与代码一致的启动方式说明。  
- **无法替代你完成**：云账号开通、付款、备案、域名解析、SSH 登录执行 `docker compose up`——这些必须在你本机/你的服务器上操作。

若你确定使用 **某一云厂商 + 某一规格实例**，可以把「系统镜像、是否有 GPU、打算用 API 还是 Gradio」发我，我可以把 `docker-compose.yml` 里的 **command/端口/卷挂载** 改成更贴近你方案的一版（仍在你仓库里维护）。
